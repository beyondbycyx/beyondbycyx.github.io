一个程序员的修炼之道

发表于 2016-12-03   |  

作为一个苦逼的自学者，也走了不少的弯路，也曾经怀疑过自己的能力，在这一路上，遇到很多的挫折，痛苦，不停的走下去之后才发现，挫折只会越来越多，痛苦只会越来越大，但是不管怎样，都在一步一步的成长中，只是这种成长的速度慢到让人无法察觉，让人失去耐心，让人失去方向。很多人都死在这条苦逼的程序员之道，每个人慢慢的失去了耐心，这时也许你需要静下心来，别把自己逼急了。过一阵子，再来看下以前学到东西，你会发现这些东西变的很简单，这就是成长的过程。我觉得这是一种每个人都与生俱来的能力—-习惯。每个人接触新的事物都会有不同的反应，有的人抵触，有的人迅速消化了解，在熟悉事物的过程中，每个人花的时间也不一样，但是每个人最后都会熟悉并习惯它，也就是说我们每个人都能成为程序员，只是时间的问题，而对于时间这是每个人都没有耐心的事情，所以编程不能，难的是能否战胜时间，活到老，学到老。

进入正题：在此总结了一下个人苦逼的自学编程之路，主要面对初学者，与各位自学者共勉

工具篇

编程需要耗费大量的时间，但是工具往往很快就能上手，好的工具能大力提升生产力，因此，对于工具必须要学的溜。

  • git 软件：

    • 介绍：开发软件必备，这是一个版本控制管理软件，最大的优点就是本地的电脑也可以当作服务器，虽然我不怎么用过svn，它的能力远远比svn强，而github就是使用了git.
    • 使用：这是一个偏向命令行的软件。因此我觉得大家要不停的敲命令行，而且要看的懂命令行字符的每一个意思，不然死记硬背会忘的更快，最好要知道它命令执行后，软件执行了什么的操作，如git init ，会在本地生成一个.git的文件夹，git commit -m ‘xxx’ ，会生成一个节点，可以使用gitk命令看下生成节点的样子。上述是使用的建议，大家还是看下官方的文档，以官方文档为准，不要看别人写了什么就是什么，比如上面我说的生成一个节点就是一个不准确的说法，这只是我的个人理解的说法。

  • android studio：

    • 介绍：这个是针对android开发的朋友来说的， 可能还有些人习惯用eclipse，但是我想说一句要跟的上时代，软件的变化，你的工具就要先更新，老是用旧的软件，只是在折磨自己。
    • 使用：
    • 第一个是了解该软件的背景： 最重要的是它是否需要其他的软件的配合支持，与其他软件的关系，比如as就用到了gradle,还有gradle的插件，还需要添加java sdk,android sdk 的路径给它。
    • 第二个看的懂软件界面的文件目录与它的作用： 每个文件名和后缀名的含义需要知道，其实看的英语多了，每个文件名或后缀名就是它的作用，特别是.xml文件，基本都是是配置信息的文件，还有不同窗口对应的功能。
    • 第三个是看下设置那里: 可设置的，添加的功能。
    • 第四个是下载安装配置一下android studio 。
    • 第五个：最重要的一点是，珍爱生命，远离鼠标，要学会精通各种快捷键，这样你的效率才会大大提高。

语言篇

  • 面向对象语言：

    • 个人推荐是先学面向对象语言类，java已经有大量的人在用了，它有很好的社区，文档，但是这几年都可以衰落的，因它是面对对象编程的，无法满足函数式编程，这也就催生了一些语言，如scala的诞生可以满足函数式编程， 能够与Rxjava 响应式函数编程很好的结合。
  • 面向过程语言：一直经久不衰的就是C 语言，虽然学过C语言，但是本人并没有做过相应的项目。

读取思想->形成思想->表达思想

读取思想->源码篇

  • 类：一个具有多种“能力”，与能够保存“数据”的代码块
    • bean类：只作为信息的载体，没有核心算法
    • util类：工具类，只起到一个工具的作用，即用即灭，并没有与其他的类有多大的联系，一般是独立的类。
    • View类：界面UI层，具有现实多种显示的效果，提供外界set方法来修改自身的样式，同时具有响应用户touch事件的功能，并内置监听器，让外界监听用户的touch事件，并回调自身。
    • wrapper类：在android 中有不少的类充当包装类的功能，如ButterInputStream就是一种，一般内部都会包裹着另一个类。
    • delegate类：委派类，android的新版本一般都是对原有的类进行增强，如Activity就是采用类委派的模式去处理各个android版本的兼容性的问题。
    • adapter类：适配器类，这也是很常见的，如View 与 Data的绑定就是需要用到了适配器。适配器模式是指将两个不同的类通过适配器结合在一起进行操作。
    • listene接口:这一般采用了回调接口模式，让外界对其进行监听。
  • 集合类：作用是存储数据，属于数据结构范围，使用场景：但数据原来越多时就是要用集合这些容器类保存，而每个容器类都有自身的存储特点以及增删特点。
    java 中：Set ,Map，Queue，Stack
    android 中：pair,lrucache,

  • 线程类：异步处理多个事件，如进行I/O的操作，网络请求的操作等，目的是提高效率

  • 同步容器类：解决线程带来的并发问题，存储多线程共享访问的数据。

  • 技巧：查看一个类的源码时，

    1. 关心它是否继承了什么类，或接口，这就代表了它具有了哪些基本的”能力”.
    2. 看这个类导入了import 哪些类，这代表了它需要那些类的帮助，有些类可能作为容器类使用，有些类可能作为工具类使用。
    3. 看下类中的基本成员变量，基本数据类型的可以忽略，可以重点关注一些集合类，容器类，通常成员变量是作为本类方法之间通信使用的。
    4. public的方法：这些方法代表提供外界使用的，这代表了该类具有的能力。
    5. private的方法：这些方法仅由本类中的其他方法来调用，由于遵循Kiss原则，每个方法只做一件事情，所以当一个方法做了太多的事情，它就开始分割多个private的方法。
    6. protected的方法：这些方法是一般留给子类使用了，让子类扩展自己的功能。
    7. 空方法：这些方法内部没有代码，通常需要覆盖重写。
      最好找优秀的开源框架或开源项目来看，学习他们的设计，写法，规范，思想。

形成思想->形成规范

  • 在这里要不停的增加自己编程世界的类，设计模式，根据看完的源代码，模仿，改造写出自己的代码,但这只是将思想写出来了,还没有到真正的表达出来,让人家看的懂的地步。创建类时需要考虑成员变量的数据结构，方法功能，必须形成良好的编码风格，习惯，和命名规范，力求过来一段时间回来看自己的代码不会排斥或看不懂

表达思想->Code Review/Refactor

  • 对已经写完的代码不停的对代码重构，改动，使他更加的“规范”，“精炼”，更加的思想化，代码尽量的简短，简练，直到别人能够一目了然，一看变量名，方法名就知道这是用来干什么的。

代码的本质—数据结构+算法

  • 去除所有的东西，包括去掉所谓的思想，只剩下主干，则剩下的东西就是数据结构和算法了。

##书籍篇

  • 《重构》：教你如何写出让别人看的懂的代码。
  • 《java的23种设计模式》：分析设计类与类之间的关系，联系，使你写出耦合性低，复用性强的代码
  • 单元测试的艺术：减少你不停找错的时间，做单元测试的好处，我觉得就是减少了出错的概率，最重要的是减少了大量找错的时间。

  • 《代码整洁之道》，《代码大全》：也是好书，我还没时间看

  • 如何看书：书可分为工具书，入门书，思想书（能看英语的就看英语，不要看翻译的，书不仅要看，还有不停的敲，特别是入门书。）

    • 工具书：这些跟文档是一样的都是作为参考，不要深读，看下就行了，需要时才看，特别是标题写着什么《100个入门案例》，还有《android 5.0 新特性》，这些谷歌官方文档都有，何必看他的，他只会将别人的文档翻译烂了，反正我是觉得都是在烂翻译。
    • 思想书：这些的书太抽象了，基本过一下，过了一段时间再看下，不停的磨它，而对于《java的23种设计模式》这样的书，我觉得最好结合java的某些源代码看，否则你看不懂，因为我就是看了很久都没结合一些常用的源代码看，所以一直看不懂。
    • 入门书：这些是每个人都要看的，基础就在这了，需要深读，将基础学好点。

注：书都是参考的，特别是软件的书，更新太快了，经典的书要读，但是还是将注意力关心在新技术上，经常上国外的网站，关注一些牛人，这才会与时俱进。很多看起来没有联系的书，如没提到的《数据结构与算法》，貌似根本不会用到，但是这些书的知识随着你的学习，工作过程中，总会在某个时空发生交集，当发生交集的时候，你就开始融会贯通了！！！
